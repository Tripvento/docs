## GET /destinations/

List all available cities with hotel counts.

### Parameters

None.

### Example request

```bash
curl -s "https://api.tripvento.com/api/v1/destinations/" \
  -H "X-Api-Key: YOUR_API_KEY"
```

### Example response

```json
{
  "count": 212,
  "results": [
    {
      "name": "Miami",
      "slug": "miami_fl",
      "label": "Miami, FL",
      "hotel_count": "150+"
    }
  ],
  "tier": "starter"
}
```

### Notes

- Sandbox and Demo tiers see 15 demo cities. All paid tiers see the full destination list.
- Use the `slug` field as the `destination` parameter in other endpoints.

## GET /rankings/

Get ranked hotels for a specific city and travel intent, ordered by `final_score` descending.

### Parameters

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `destination` | string | Yes | Destination slug (e.g. `miami_fl`, `aspen_co`). Preferred over `city`. |
| `city` | string | No | City name (e.g. `Miami`). Deprecated — use `destination` instead. |
| `intent` | string | Yes | Travel intent slug (see [Travel Intents](/intents)). |
| `thin` | boolean | No | Strip hotel detail fields. Returns `id`, `name`, `location`, `detail_url` only alongside full scores. |

### Example request

```bash
curl -s "https://api.tripvento.com/api/v1/rankings/?destination=miami_fl&intent=romantic" \
  -H "X-Api-Key: YOUR_API_KEY"
```

## GET /intents/{id}/

Full scoring detail for a single hotel/intent pair — the complete explanation behind a ranking.

Get the `id` from the `detail_url` in city-matrix's `intent_matrix` object, or from rankings results.

### Example request

```bash
curl -s "https://api.tripvento.com/api/v1/intents/98123/" \
  -H "X-Api-Key: YOUR_API_KEY"
```

### Example response

```json
{
  "id": 98123,
  "intent_name": "romantic",
  "final_score": 88.5,
  "geo_score": 91.2,
  "ai_score": 85.3,
  "sentiment_score": 89.1,
  "neighborhood_vibe_score": 82.0,
  "editorial_score": null,
```

## GET /city-matrix/

All hotels in a city with scores across every travel intent. One request replaces 14 individual ranking calls.

<Note>Requires **Startup tier** or above. Starter and Sandbox tiers receive a 403 response.</Note>

### Parameters

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `destination` | string | Yes | Destination slug (e.g. `miami_fl`). Preferred over `city`. |
| `city` | string | No | City name. Deprecated — use `destination` instead. |
| `thin` | boolean | No | Default `true`. Strip hotel detail fields, keep full intent matrix scores. Pass `false` to include full hotel details. |

### Example request

```bash
curl -s "https://api.tripvento.com/api/v1/city-matrix/?destination=miami_fl" \
  -H "X-Api-Key: YOUR_API_KEY"
```

### Example response (default thin)

```json
{

    {
      "id": 1234,
      "name": "Faena Hotel Miami Beach",
      "detail_url": "https://api.tripvento.com/api/v1/hotels/1234/",

      "intent_matrix": {
        "romantic": {
          "id": 98123,
          "detail_url": "https://api.tripvento.com/api/v1/intents/98123/",
          "final_score": 88.5,
          "geo_score": 91.2,
          "ai_score": 85.3,
          "sentiment_score": 89.1,
          "vibe_score": 82.0,
          "editorial_score": 0.0
        },
        "business_road_warrior": {
          "id": 98124,
          "detail_url": "https://api.tripvento.com/api/v1/intents/98124/",
          "final_score": 62.1,
```

### Notes

- Best for building comparison pages, search filters, or pre-caching an entire city.
- Use `detail_url` on each intent to fetch the full breakdown on demand.

## GET /hotels/{id}/

Hotel metadata without ranking scores. Use `/rankings/` or `/city-matrix/` for scored results.

### Parameters

| Parameter | Type | Required | Description |
| --------- | ---- | -------- | ----------- |
| `city` | string | No | Filter by city name. |
| `destination` | string | No | Filter by destination slug. |

### Example request

```bash
curl -s "https://api.tripvento.com/api/v1/hotels/1234/" \
  -H "X-Api-Key: YOUR_API_KEY"
```

## GET /usage/

Check your current quota and usage. This call does not count toward your monthly limit.

### Parameters

None.

### Example request

```bash
curl -s "https://api.tripvento.com/api/v1/usage/" \
  -H "X-Api-Key: YOUR_API_KEY"
```

### Example response

```json
{
  "tier": "starter",
  "monthly_limit": 5000,
  "monthly_used": 127,
  "monthly_remaining": 4873,
  "burst_limit_per_minute": 120,
  "resets": "2026-03-01",
  "unlimited": false
}
```

## Error responses

See [Errors](/errors) for the full reference.

| Status | Meaning |
| ------ | ------- |
| 400 | Bad request — missing or invalid parameters |
| 403 | Forbidden — invalid or missing API key, or tier restriction |
| 404 | Not found — city or resource does not exist |
| 429 | Rate limited — monthly or burst limit exceeded |
| 500 | Server error — try again or contact support |
